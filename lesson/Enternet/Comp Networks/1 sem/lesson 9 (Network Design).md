# Проектирование сети

### Конвергентные сети.

Конвергентные сети позволяют передавать данные, голос и видео через одну сеть, исключая необходимость в отдельных
каналах для каждого типа трафика. Это современный подход к сетевой инфраструктуре, обеспечивающий экономию ресурсов.

**Преимущества:** Единая сеть упрощает управление и снижает затраты на оборудование и обслуживание.

**Недостатки:** Требует продуманной архитектуры для предотвращения задержек и искажений, что повышает сложность
проектирования.

**Требования:** Обеспечение низкой задержки, высокая надежность, поддержка критических приложений, масштабируемость для
роста сети.

### Выбор оборудования при построении сети.

**Коммутаторы:**

* Фиксированная Конфигурация: Ограниченные функции, например, Cisco 3850 (12-48 портов), подходит для небольших сетей.
* Модульная Конфигурация: Высокая плотность портов (до 384 на Catalyst 9400), идеальна для крупных сетей.
* Стекируемая Конфигурация: Единое управление через специальные кабели, упрощает администрирование.
* По Применению: Meraki (облачное управление), Nexus (ЦОД), операторские коммутаторы.

**Характеристики:**

* Буфер порта: Предотвращает заторы, важен для загруженных сетей.
* Плотность портов: Определяет количество подключенных устройств.
* PoE: Поддержка питания по Ethernet (IP-телефоны, точки доступа).
* Скорость: 100 Мбит/с, 1/10/100 Гбит/с в зависимости от потребностей.
* Надежность и масштабируемость: Критичны для роста сети.

**Маршрутизаторы:**
*Филиальные: Cisco 900 для оптимизации сервисов в филиалах.
*Граничные: ASR 9000/1000 для периметра сети.
*Промышленные: Cisco 800 для суровых условий.

**Функции:**

* Маршрутизация
* фильтрация трафика
* резервирование путей.
